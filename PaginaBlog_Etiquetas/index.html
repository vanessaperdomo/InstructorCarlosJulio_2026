Fundamentos de HTML

La base fundamental del desarrollo web moderno.

¿Qué es HTML?

HTML es la sigla de HyperText Markup Language, que en español significa Lenguaje de Marcado de Hipertexto. Es el lenguaje estándar utilizado para la creación y estructuración de páginas web.

HTML permite organizar y mostrar el contenido mediante etiquetas o tags, las cuales indican al navegador cómo debe mostrarse la información (títulos, párrafos, imágenes, formularios, etc.). Los navegadores interpretan este código y lo convierten en la página visual que ves en internet.

¿Qué significa HTML?

H

HyperText

Capacidad de conectar documentos mediante hipervínculos (links), permitiendo navegar entre diferentes páginas.

M

Markup

Uso de etiquetas para marcar y estructurar el contenido dentro de un documento.

L

Language

Conjunto de reglas y sintaxis utilizadas para escribir y organizar el contenido web.

¿Para qué sirve HTML?

Se utiliza para crear la estructura de las páginas web. Sus funciones principales son:

  • Definir la estructura básica y jerarquía.
  • Crear títulos y párrafos de texto.
  • Insertar imágenes y contenido multimedia (videos/audios).
  • Crear enlaces y organizar datos en tablas o listas.

HTML trabaja junto a:

🎨 CSS: Estilo y diseño.

⚡ JavaScript: Interactividad y dinámica.

✅ Ventajas

  • Fácil de aprender: Sintaxis clara para principiantes.
  • Compatibilidad: Funciona en todos los navegadores modernos.
  • Gratuito y Abierto: Sin costo de licencias.
  • Base Universal: Todos los sitios web lo utilizan.
  • Integración: Se combina con CSS y JS.

❌ Desventajas

  • No es programación: No ejecuta lógica compleja por sí solo.
  • Páginas Estáticas: Sin interactividad sin JavaScript.
  • Visualmente Pobre: Necesita CSS para verse bien.
  • Limitación de Datos: No se conecta directo a bases de datos.

Conclusión

HTML es el corazón de la web. Aunque tiene limitaciones, su facilidad de uso y capacidad de integración lo convierten en una herramienta esencial para cualquier desarrollador web profesional.

Etiquetas en HTML

Los ladrillos que construyen la estructura semántica de la web.

¿Qué son las etiquetas?

Las etiquetas HTML son elementos utilizados para definir y estructurar el contenido de una página web. Indican al navegador cómo debe interpretarse y mostrarse cada parte del contenido (textos, imágenes, tablas, etc.).

Están formadas por palabras clave encerradas entre signos de menor y mayor que < >. Generalmente se componen de una apertura y un cierre.

<p> Este es un párrafo </p>
Apertura Contenido Cierre

¿Para qué sirven?

1

Organización

Permiten que los navegadores interpreten el contenido correctamente (títulos, listas, videos).

2

Jerarquía

Definen qué información es más importante mediante encabezados y secciones.

3

Interactividad

Facilitan la creación de formularios y enlaces para conectar con el usuario.

¿Cómo se usan?

Se escriben directamente en el documento. La mayoría sigue la estructura básica: <etiqueta>contenido</etiqueta>.

<h1>Título principal</h1>
<p>Este es un párrafo de texto.</p>

Además, pueden incluir atributos (propiedades adicionales):

<img src="imagen.jpg" alt="Descripción" width="200">
  • src: ruta del archivo.
  • alt: descripción para accesibilidad.
  • width: define el tamaño.

✅ Ventajas

  • Fácil de aprender: Nombres simples y sintaxis lógica.
  • Organización: Estructura clara y ordenada.
  • Compatibilidad: Funcionan en todos los navegadores.
  • Base Universal: Estándar mundial de la web.
  • Integración: Se combinan perfecto con CSS y JS.

❌ Desventajas

  • Sin lógica: No realizan cálculos ni procesos complejos.
  • Dependencia: Requieren CSS para diseño avanzado.
  • Mantenimiento: El código puede ser extenso en sitios grandes.
  • Datos: No se conectan directo a bases de datos.

Categorías de etiquetas

Se clasifican según su función:

Etiquetas estructurales Etiquetas de texto Etiquetas de listas Etiquetas de tablas Etiquetas de formularios Etiquetas multimedia

Resumen Educativo

Las etiquetas son los ladrillos de la web. Sin ellas, el navegador no sabría diferenciar entre un simple texto y un botón interactivo. Dominarlas es el primer paso para ser un desarrollador profesional.

1. Etiquetas Estructurales

Estas etiquetas definen la arquitectura semántica y jerarquía del documento web.

Etiqueta: header

Definición

La etiqueta header representa la cabecera de una página o de una sección específica. Suele contener logotipos, títulos de navegación o formularios de búsqueda.

Sintaxis basica / demostracion visual

Uso Adecuado

Se utiliza para introducir el contenido que le sigue o para facilitar la navegación global del sitio.

Buenas prácticas:
  • No debe confundirse con la etiqueta head.
  • Puede haber varios header en una página (uno en el body y otros dentro de section o article).
Etiqueta: section

Definición

Define una sección genérica y autónoma de contenido dentro del documento, la cual no tiene un contenedor más específico que la represente.

Sintaxis básica / demostraccion visual

Uso Adecuado

Se emplea para agrupar contenido que comparte una misma temática, ayudando a los lectores de pantalla a entender la división del sitio.

Buenas prácticas:
  • Cada sección debería incluir, por norma general, un encabezado (h2-h6).
  • No usar como un simple contenedor de diseño (para eso existe el div).
Etiqueta: footer

Definición

Representa el pie de página de un documento o de una sección. Suele contener información sobre el autor, datos de copyright o enlaces a redes sociales.

Sintaxis básica / demostracion visual

Uso Adecuado

Se coloca al final del contenido del cual es "pie", proporcionando información de cierre o metadatos del autor.

Buenas prácticas:
  • Mantenerlo simple y con información legal o de contacto relevante.
  • Al igual que el header, puede repetirse en diferentes secciones.

2. Etiquetas Texto

Elementos para estructurar informacion escrita y dar jerarquia al contenido.

Etiqueta: h1

Definicion

Representa el titulo principal del documento. Es el nivel mas alto de jerarquia.

Sintaxis / Visual

(h1) Este es el titulo principal (/h1)

Uso Adecuado

Se usa para el tema central de la pagina. Ayuda a los motores de busqueda a indexar el sitio.

Buenas practicas:
  • Usar solo un (h1) por pagina web.
  • No usar para hacer texto grande, para eso se usa CSS.
Etiqueta: p

Definicion

Define un parrafo de texto. El navegador añade automaticamente espacio antes y despues.

Sintaxis / Visual

(p) Este es un bloque de texto informativo (/p)

Uso Adecuado

Ideal para bloques de lectura. Agrupa oraciones relacionadas con un mismo tema.

Buenas practicas:
  • Mantener parrafos cortos para facilitar la lectura.
  • No meter etiquetas de bloque como (div) dentro de un (p).
Etiqueta: strong

Definicion

Indica que el texto tiene una importancia fuerte, seriedad o urgencia.

Sintaxis / Visual

Texto normal (strong) texto importante (/strong)

Uso Adecuado

Muestra el texto en negrita por defecto, pero su valor principal es avisar a lectores de pantalla que es contenido vital.

Buenas practicas:
  • No abusar de las negritas en todo el texto.
  • Usar solo en palabras clave o advertencias.

3. Etiquetas de Listas

Elementos para organizar informacion en puntos, secuencias o glosarios.

Etiqueta: ul

Definicion

Crea listas desordenadas, donde el orden no es relevante. Los items aparecen con viñetas.

(ul) ... (/ul)

Uso Adecuado

Ideal para listas de compras, caracteristicas o menus de navegacion simple.

Buenas practicas:
  • Solo debe contener etiquetas (li) como hijos directos.
Etiqueta: ol

Definicion

Crea listas ordenadas. Los elementos siguen una secuencia numerica o alfabetica automatica.

(ol) ... (/ol)

Uso Adecuado

Se usa para instrucciones, pasos de una receta o cualquier contenido con orden jerarquico.

Etiqueta: li

Definicion

Define un elemento individual dentro de una lista (ul) o (ol).

(li) Item de lista (/li)

Uso Adecuado

Representa cada punto de la lista. No debe usarse solo, siempre dentro de un contenedor de lista.

Etiqueta: dl

Definicion

Representa una lista de descripcion o glosario, conectando terminos con sus explicaciones.

(dl) (dt)Termino(/dt) (dd)Descripcion(/dd) (/dl)

Uso Adecuado

Se usa para diccionarios, preguntas frecuentes (FAQ) o metadatos de un articulo.

Estructura interna:
  • (dt): El termino a definir.
  • (dd): La definicion o descripcion del termino.

4. Etiquetas de Formularios

Elementos para interactuar con el usuario y recopilar datos de entrada.

Etiqueta: form

Definicion

Contenedor principal que agrupa campos de entrada para enviar informacion al servidor.

(form) ... (/form)

Uso Adecuado

Se usa para crear secciones de contacto, registros o inicios de sesion.

Buenas practicas:
  • Incluir etiquetas descriptivas para cada campo.
  • Validar que los datos sean correctos antes de enviar.
Etiqueta: input

Definicion

Campo versatil donde el usuario introduce datos. No necesita etiqueta de cierre.

(input type="text" /)

Uso Adecuado

Segun su "type", puede ser texto, correo, contraseña, numero o incluso un selector de fecha.

Buenas practicas:
  • Usar el tipo de entrada adecuado (email para correos, etc).
  • Usar el atributo placeholder para dar pistas al usuario.
Etiqueta: label

Definicion

Define un nombre o etiqueta de texto vinculada a un campo de entrada (input).

(label) Nombre: (/label)

Uso Adecuado

Mejora la accesibilidad. Al hacer clic en el texto, el cursor se posiciona en el campo asociado.

Etiqueta: button

Definicion

Crea un boton clickeable para ejecutar acciones o enviar el formulario.

(button) Enviar Datos (/button)

Uso Adecuado

Se usa para procesar el formulario o realizar acciones interactivas con JavaScript.

Buenas practicas:
  • Usar textos claros que indiquen la accion (ej: "Registrarse").
  • Indicar el tipo de boton (type="submit" o "button").

5. Etiquetas de Tablas

Elementos para organizar y presentar datos estructurados en filas y columnas.

Etiqueta: table

Definicion

Contenedor principal que define una tabla para organizar informacion compleja.

(table) ... (/table)

Uso Adecuado

Se utiliza para mostrar datos tabulares, como listas de precios, horarios o resultados.

Buenas practicas:
  • Usar tablas solo para datos estructurados, nunca para maquetar el diseño del sitio.
Etiqueta: tr

Definicion

Define una fila (table row) dentro de la tabla que agrupara las celdas de datos o encabezados.

(tr) ... (/tr)

Uso Adecuado

Organiza la informacion de forma horizontal. Cada fila representa un registro de la tabla.

Etiqueta: th

Definicion

Define una celda de encabezado (table header). El texto suele aparecer centrado y en negrita.

(th) Titulo de Columna (/th)

Uso Adecuado

Se usa en la primera fila o columna para identificar que tipo de datos siguen a continuacion.

Etiqueta: td

Definicion

Define una celda de datos estandar (table data) dentro de una fila de la tabla.

(td) Dato especifico (/td)

Uso Adecuado

Contiene la informacion real (texto, numeros, imagenes) que se desea mostrar en la cuadricula.

Buenas practicas:
  • Mantener el contenido claro y alineado correctamente con su encabezado.

6. Etiquetas Multimedia

Elementos para integrar contenido visual y auditivo de forma interactiva.

Etiqueta: img

Definicion

Se utiliza para insertar imagenes. Es una etiqueta de cierre automatico (no necesita etiqueta de cierre).

(img src="ruta/imagen.jpg" alt="descripcion" /)

Uso Adecuado

Agrega contenido visual. Requiere el atributo src para la ruta y alt para accesibilidad.

Buenas practicas:
  • Siempre usar el atributo alt para lectores de pantalla.
  • Optimizar el tamaño de la imagen para mejorar la carga.
Etiqueta: audio

Definicion

Permite insertar archivos de sonido directamente en el navegador sin plugins externos.

Uso Adecuado

Ideal para podcasts, musica o sonidos educativos. El atributo "controls" es vital para que el usuario pueda darle play.

Buenas practicas:
  • Ofrecer controles de reproduccion claros.
  • Evitar la reproduccion automatica (autoplay) para no molestar al usuario.
Etiqueta: video

Definicion

Permite insertar videos nativamente. Es compatible con formatos modernos como MP4, WebM y Ogg.

Uso Adecuado

Se usa para mostrar tutoriales o contenido audiovisual. Permite definir ancho (width) y alto (height) directamente.

Buenas practicas:
  • Incluir siempre el atributo controls para que el usuario gestione el volumen y tiempo.
  • Usar el atributo poster para mostrar una imagen previa antes de iniciar el video.
Etiqueta: source

Definicion

Se utiliza dentro de (audio) o (video) para especificar multiples archivos de origen o diferentes formatos.

(source src="video.mp4" type="video/mp4" /)

Uso Adecuado

Sirve para que el navegador elija el formato que mejor soporte segun su compatibilidad interna.

Buenas practicas:
  • Colocar los formatos mas ligeros y compatibles (como MP4) primero en la lista.
